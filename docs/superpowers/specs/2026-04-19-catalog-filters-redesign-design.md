# Редизайн фильтров каталога

Дата: 2026-04-19
Область: `frontend/src/pages/Catalog.tsx` + стили в `frontend/src/index.css` (классы `.zen-filters-*`)

## Проблема

Текущая шторка фильтров (см. референс-скриншот `assets/______________2026-04-19___13.16.30-*.png`):

1. Визуально «дырявая»: под заголовком большая пустая область, заголовки секций «ЦЕНА», «КАТЕГОРИИ» с маленькой стрелкой `▶` выглядят как пункты меню, а не как аккордеон.
2. Маленькая стрелка-шеврон вниз в футере — непонятное действие. Нет главной кнопки.
3. Секция «Бренд» пропала в текущей сборке — потому что в условии `uniqueBrands.length > 0` у товаров каталога не заполнено поле `brand`. Секция внезапно исчезает, хотя пользователь мог ожидать её увидеть.
4. Активные значения фильтров показаны лентой тегов под заголовком («panel-summary»), которая визуально отделена от секций и занимает вертикальное место даже когда фильтров мало.

## Итог брейншторма (выборы пользователя)

- **Направление:** Теги + фасеты (вариант D из `filter-directions.html`).
- **Поведение раскрытия секций:** `D1 · Inline accordion` — тап по строке фасета раскрывает её in-place, остальные фасеты остаются видны ниже.
- **Активные значения:** `C · Inline в строке фасета` — никакой отдельной ленты; активные значения — маленькие пилюли с «×» прямо в строке своей фасеты (справа от названия секции).
- **Секция «Бренд»:** `Option 3` — показываем только когда `uniqueBrands.length >= 2`.
- **Футер:** `F2` — главная кнопка «Показать N товаров» с черновым состоянием фильтров.
- **Закрытие шторки:** `X2` — любое закрытие (✕, фон, свайп вниз) применяет текущий черновик. Кнопки «Отмена» нет.

### Раскрытая секция

Когда пользователь тапает на строку фасета, под ней появляется блок `.zen-filters-facet-expanded` с фоном `#faf7f1`. Стрелка `▸` поворачивается на 90° → `▾`. Другие фасеты не сворачиваются автоматически — пользователь сам решает, закрыть ли предыдущую.

**Цена (раскрыто):**

- Двухручечный слайдер (используем существующий `.zen-filters-price-slider*`).
- Подписи min/max значений по концам.
- Segmented control «↑ По возрастанию» / «↓ По убыванию» (используем существующий `.zen-filters-sort-segmented`).

**Бренд (раскрыто):**

- Строка чипов: «Все» + по одному чипу на бренд. Single-select (одно значение за раз, как сейчас).
- Использует существующие стили `.zen-filters-chip`, `.zen-filters-chip-active`.

**Категории (раскрыто):**

- Строка чипов категорий. Multi-select (как сейчас).

### Поведение

#### Черновое состояние (F2 + X2)

При открытии шторки текущие применённые фильтры копируются в локальные draft-стейты:

- `draftPriceMin`, `draftPriceMax`, `draftPriceSort`
- `draftSelectedBrand`
- `draftSelectedCategories`

Все контролы внутри шторки пишут в draft. Каталог снаружи перерисовывается ТОЛЬКО при «применении».

**Применение** (`commitDraft()`):

1. Копируем draft → applied state (`priceMin`, `priceMax`, `priceSort`, `selectedBrand`, `selectedCategories`).
2. Закрываем шторку (`closeFilters()`).

**Триггеры применения:**

- Тап по кнопке «Показать N товаров».
- Тап по «✕» в шапке шторки.
- Тап по фону оверлея.
- Свайп вниз за drag-handle (существующий механизм).
- Нажатие Escape.

Т.е. ЛЮБОЙ способ закрытия = commit. Нет пути «закрыл без применения».

**Сброс** (`resetDraft()`):

- Кнопка «Сброс» в футере сбрасывает все draft-поля к «all» / пустым.
- Кнопка в футере **видна только когда в draft есть непустые значения** (иначе скрываем — на пустом состоянии видна только большая кнопка «Показать N»).
- Сброс НЕ закрывает шторку — пользователь видит чистое состояние и может применить его (закрытием / кнопкой), либо выбрать новые фильтры.

#### Счётчик «Показать N товаров»

`N` = число товаров, которое получилось бы при применении текущего DRAFT. Считается тем же алгоритмом, что и `filtered`, но на draft-значениях. Обновляется реактивно на каждый тап по чипу / движение слайдера.

Если `N === 0` — кнопка показывает «Ничего не найдено» и становится полупрозрачной (но остаётся кликабельной — нажатие всё равно применяет draft и закрывает шторку; пользователь увидит пустой каталог и сможет откатить).

#### Активные чипы inline

В свёрнутой фасет-строке справа от названия секции идёт контейнер `.zen-filters-facet-values`:

- Если в этой секции в draft ничего не выбрано — показываем текст `.zen-filters-facet-empty` = «все» (mute, полупрозрачный).
- Если есть выбранные значения — показываем их как маленькие пилюли `.zen-filters-facet-chip`:
  - Цена: одна пилюля `{min} — {max} ₽` (или `до {max} ₽`, или `от {min} ₽`). Нажатие на «×» чипа сбрасывает цену.
  - Бренд: одна пилюля с названием бренда.
  - Категории: по пилюле на каждую выбранную категорию.
- Клик по «×» пилюли — сбрасывает именно это значение и НЕ раскрывает секцию.
- Клик по самой строке фасеты (но не по пилюле) — раскрывает/сворачивает секцию.

#### Секция «Бренд»

Условие показа: `uniqueBrands.length >= 2`. Если только 0 или 1 бренд — секция не рендерится (фильтровать нечего). При этом draft-значение `selectedBrand` всё равно поддерживается в состоянии (не сбрасывается принудительно), чтобы при возврате товаров с брендами пользователь не терял выбор.

## Архитектура компонента

### Границы

Редизайн локализован в `Catalog.tsx`. Чтобы файл не разросся ещё больше, выносим шторку в отдельный компонент:

**Новый файл:** `frontend/src/components/catalog/FiltersSheet.tsx`

Интерфейс:

```ts
interface FiltersSheetProps {
  open: boolean;
  closing: boolean;
  onAnimationEnd: () => void;
  onClose: () => void; // триггерит commit + анимацию закрытия

  // applied values — для инициализации draft при open
  appliedPriceMin: string;
  appliedPriceMax: string;
  appliedPriceSort: "none" | "asc" | "desc";
  appliedBrand: string;
  appliedCategories: Set<string>;

  // данные
  uniqueBrands: string[];           // если < 2 — секция скрывается внутри
  categoryTabs: { code: string; label: string }[];
  catalogPriceMin: number;
  catalogPriceMax: number;

  // подсчёт превью-N
  countForDraft: (draft: DraftFiltersValue) => number;

  // commit
  onApply: (draft: DraftFiltersValue) => void;

  lang: "ru" | "en";
}

interface DraftFiltersValue {
  priceMin: string;
  priceMax: string;
  priceSort: "none" | "asc" | "desc";
  brand: string;
  categories: Set<string>;
}
```

`Catalog.tsx` отвечает за:
- applied state (как сейчас),
- открытие/закрытие шторки,
- передачу функции `countForDraft`, которая использует тот же алгоритм, что и `filtered`, но на draft-аргументах.

`FiltersSheet.tsx` отвечает за:
- локальный draft,
- DOM шторки, рендер фасет, раскрытие секций,
- слайдер цены и его drag-логика (перенести из текущего Catalog),
- handle drag-to-close,
- pointer events для фона/Escape,
- вызов `onApply(draft)` при любом триггере закрытия.

### Data flow

```
Catalog (applied) ──open──▶ FiltersSheet
                             ├─ инициализирует draft = applied
                             ├─ пользователь меняет draft
                             ├─ countForDraft(draft) → N
                             ├─ при close: onApply(draft)
                             └─ FiltersSheet unmount
Catalog.applied ◀─setApplied── onApply
Catalog.filtered ← applied    (как сейчас)
```

## Тестирование

Так как это UI-компонент с взаимодействием, проверяем руками по сценариям:

1. Открытие шторки — draft инициализируется текущими applied значениями; счётчик на кнопке = текущее число товаров.
2. Изменение цены слайдером — кнопка «Показать N» обновляется, но каталог под шторкой НЕ меняется.
3. Тап «Показать N» — шторка закрывается, каталог показывает N товаров.
4. Тап «✕» в шапке — draft применяется, каталог обновляется.
5. Свайп шторки вниз — draft применяется.
6. Тап по фону — draft применяется.
7. Escape — draft применяется.
8. Тап «Сброс» — все чипы в draft сбрасываются, счётчик «Показать N» становится общим числом товаров, кнопка «Сброс» исчезает. Шторка НЕ закрывается.
9. Тап на «×» в активной пилюле фасеты — сбрасывается это значение, секция не раскрывается.
10. Тап в пустую область строки фасеты — раскрывается секция.
11. Тап по уже раскрытой строке — секция сворачивается.
12. У товаров каталога нет `brand` ни у одного — секция «Бренд» не отображается.
13. У товаров каталога только 1 уникальный `brand` — секция «Бренд» не отображается.
14. У товаров каталога ≥ 2 разных `brand` — секция отображается, после выбора бренда каталог фильтруется, activeChip появляется в строке фасеты.
15. N = 0 в draft — кнопка «Ничего не найдено», полупрозрачная, но кликабельная.
16. Переключение языка RU/EN в открытой шторке — все надписи локализованы.
17. Drag-handle работает как раньше (свайп вниз закрывает).

Автотестов на UI в проекте нет, проверка ручная.

3. Переписать разметку — фасеты вместо секций-аккордеона, пилюли активных значений inline.
4. Добавить новые CSS-классы в `index.css`, удалить устаревшие (`.zen-filters-panel-summary`, `.zen-filters-panel-close-arrow`).
5. В `Catalog.tsx` убрать текущую разметку шторки (≈ строки 605–810), заменить на `<FiltersSheet ... />`.
6. Оставить applied-state в `Catalog.tsx`, убрать draft-логику из него (если она где-то смешана).
7. Добавить `countForDraft` функцию (чистая функция, повторяющая логику `filtered`).
8. Добавить ключи локализации `filtersShowN`, `filtersNothingFound`.
9. Проверить ручными сценариями выше.

## Не в scope

- Серверные изменения (API `Product.brand` и админка остаются как есть).
- Массовое заполнение брендов у товаров — это задача контент-менеджера в админке, не этот спек.
- Дополнительные фасеты (цвет, размер, материал) — в будущем легко добавляются как ещё одна `.zen-filters-facet`, но сейчас не нужны.
- Сохранение фильтров в URL/localStorage.
- Анимация поворота стрелки > простого CSS transition (FLIP, motion и т.п.).
