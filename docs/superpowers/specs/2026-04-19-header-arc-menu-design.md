# Arc Menu в хедере — дизайн

## Контекст

Сейчас при клике на бургер в хедере (`frontend/src/App.tsx`, кнопка `zen-header-hamburger`) выезжает боковая панель (`styles.menu`) с затемнением фона (`styles.menuOverlay`) и четырьмя пунктами списком: Профиль, История, Отзывы, Настройки.

Нужно заменить это на анимацию, где при клике на бургер те же четыре пункта «разлетаются» в виде круглых иконок по дуге вокруг самой кнопки.

## Цели

- Сохранить те же четыре действия и их текущие обработчики (`openProfile`, `openHistory`, `openReviews`, `openSettings`).
- Заменить боковую панель на радиальное arc-меню.
- Держать визуальный язык в рамках зен-эстетики проекта (спокойно, без пружин, тонкие обводки, glass-эффект как в `BottomNavBar`).
- Сохранить доступность (aria-labels, закрытие по клику вне, закрытие по выбору пункта).

## Решения

### Геометрия

- **Форма:** четверть круга вниз-вправо от центра кнопки бургера.
- **Радиус:** 90px от центра бургера до центра каждой иконки-кружка.
- **Углы:** четыре позиции, равномерно распределённые по дуге 0°–90°, где 0° — строго вниз, 90° — строго вправо. Точные углы: 17°, 39°, 61°, 83° (равномерный шаг ≈22°, с отступом от краёв, чтобы первый и последний кружок не совпадали с осями).
- **Порядок по дуге (от горизонтали к вертикали):**
  1. Профиль — ~83° (сверху-справа, ближе к горизонтали)
  2. История — ~61°
  3. Отзывы — ~39°
  4. Настройки — ~17° (снизу, ближе к вертикали)

Центр отсчёта — центр кнопки бургера. Иконки позиционируются `position: absolute` относительно обёртки, совпадающей по координатам с бургером.

### Анимация

- **Поведение:** все четыре кружка стартуют одновременно из позиции бургера (scale 0.6, opacity 0), плавно летят на свои позиции по дуге и проявляются.
- **Длительность:** 500ms.
- **Easing:** `cubic-bezier(0.22, 1, 0.36, 1)` — плавный ease-out без пружины.
- **Fade:** `opacity` меняется параллельно с `transform`, длительность 350ms, тот же easing.
- **Закрытие:** тот же переход в обратную сторону (кружки возвращаются в точку бургера и исчезают). После окончания анимации они убираются из DOM / получают `pointer-events: none`.

### Внешний вид кружков

- **Размер:** 40×40px, `border-radius: 50%`.
- **Фон:** `rgba(var(--bg-rgb), 0.75)`.
- **Blur:** `backdrop-filter: blur(10px)` + `-webkit-backdrop-filter: blur(10px)`.
- **Обводка:** `1px solid rgba(0,0,0,0.08)` (в тёмной теме — `rgba(255,255,255,0.08)`, если `--border` не подходит — берём `var(--border)`).
- **Тень:** `box-shadow: 0 2px 8px rgba(0,0,0,0.06)` — очень лёгкая, для отделения от фона.
- **Иконка внутри:** те же SVG-компоненты, что сейчас (`MenuIconProfile`, `MenuIconHistory`, `MenuIconReviews`, `MenuIconSettings`), размер 20×20px, цвет `var(--text)`.
- **Рейтинг отзывов:** текстовый рейтинг `★ 4.8`, который сейчас отображается рядом с пунктом «Отзывы» в боковой панели, в arc-меню **не показывается**. Рейтинг видно только на самой странице отзывов.

### Overlay

- **Тип:** прозрачный «ловец кликов» (`position: fixed, inset: 0, background: transparent`).
- **Поведение:** клик по нему закрывает меню. Фон каталога остаётся видимым и не затемняется.
- **z-index:** между контентом страницы и иконками меню, чтобы иконки принимали клики, а остальной экран — нет.

### Состояние бургера

- При открытом меню бургер поворачивается на `90°` (transform: rotate(90deg)).
- Цвет меняется с `var(--text)` на `var(--accent)`.
- Transition: 350ms ease (совпадает с fade кружков).
- Сам SVG не меняется — только поворот контейнера-кнопки.

### Закрытие меню

Меню закрывается:
1. Повторным кликом на бургер.
2. Кликом по overlay-ловцу (вне кружков).
3. Выбором любого пункта меню (текущие обработчики `openProfile` и т.д. уже делают `setMenuOpen(false)`).
4. По `Escape` (новое — для клавиатурной доступности).

## Архитектура

### Компоненты

Выделить новый компонент `HeaderArcMenu` в `frontend/src/components/HeaderArcMenu.tsx`:

```
interface HeaderArcMenuProps {
  open: boolean;
  onClose: () => void;
  onProfile: () => void;
  onHistory: () => void;
  onReviews: () => void;
  onSettings: () => void;
}
```

Компонент рендерит:
- Прозрачный overlay (только когда `open === true`).
- Контейнер иконок, расположенный абсолютно в том же месте, что и бургер (фиксирован к левому верхнему углу хедера).
- Четыре кнопки-кружка с иконками.

Сами SVG-иконки (`MenuIconProfile` и т.д.) переносятся из `App.tsx` в `HeaderArcMenu.tsx` или в отдельный файл `MenuIcons.tsx` — они больше нигде не используются.

### Изменения в `App.tsx`

- Оставить состояние `menuOpen` и все `openProfile/openHistory/openReviews/openSettings` как есть.
- Удалить блок `{menuOpen && (<>...overlay + panel...</>)}` (строки ~332–365).
- Удалить стили `menuOverlay`, `menu`, `menuItem`, `menuItemContent` из объекта `styles` (если они больше нигде не используются).
- Удалить компоненты `MenuIconProfile/History/Reviews/Settings` из `App.tsx` (переехали в новый файл).
- Вставить `<HeaderArcMenu open={menuOpen} onClose={() => setMenuOpen(false)} ... />` сразу после `<header>`.

### Позиционирование

Бургер сейчас находится внутри `<header>` с `position: sticky` (вероятно) и паддингами. Кружки должны быть позиционированы относительно **самого бургера**, а не хедера. Два варианта реализации:

1. **Portal + ref:** `HeaderArcMenu` получает ref на кнопку бургера, вычисляет её позицию через `getBoundingClientRect()`, рендерит кружки в `position: fixed` с нужными координатами.
2. **Вложенный контейнер:** arc-меню рендерится как дочерний элемент `<header>`, а иконки позиционируются `position: absolute` относительно бургера через CSS (бургер получает `position: relative`, иконки — `position: absolute; top: 50%; left: 50%` + transform).

**Выбор — вариант 2** (вложенный контейнер). Проще, не требует вычислений координат, естественно реагирует на ресайз. Бургеру добавляется `position: relative`, внутри него рендерится абсолютно-позиционированный слой с иконками, который «вылетает» за границы бургера (overflow не режется).

## Data flow

```
User clicks burger
  → App.setMenuOpen(true)
  → HeaderArcMenu receives open=true
  → CSS class `.open` применяется к контейнеру
  → все 4 кружка одновременно анимируются на свои позиции

User clicks icon
  → onProfile/onHistory/... в App
  → setMenuOpen(false) внутри этих функций (уже есть)
  → переход на новую страницу
  → HeaderArcMenu receives open=false → кружки анимируются обратно

User clicks overlay / burger / presses Esc
  → onClose() → setMenuOpen(false)
  → кружки анимируются обратно
```

## Обработка ошибок / edge cases

- **Быстрые клики открыть-закрыть:** CSS transitions сами обрабатывают — прерывают текущую анимацию и начинают новую с текущей позиции. Отдельного кода не нужно.
- **Backdrop-filter не поддерживается (старые браузеры):** fallback — фон кружка становится `var(--bg)` без прозрачности. Включается через `@supports not (backdrop-filter: blur(10px))`.
- **Prefers-reduced-motion:** если у пользователя включено, transitions сокращаются до 0ms, и кружки просто появляются/исчезают без движения. Реализуется через media-query.
- **RTL-локали:** в проекте RTL пока не используется (судя по `i18n.ts`). Оставляем hardcoded «вправо-вниз». Если в будущем появится RTL — нужно будет зеркалить.

## Тестирование

Unit-тесты не нужны — это чисто визуально-анимационный компонент без логики. Проверяем руками:

1. Клик на бургер открывает меню; все 4 кружка появляются плавно.
2. Клик ещё раз на бургер — закрывает.
3. Клик на любой из кружков — выполняет своё действие и закрывает меню.
4. Клик вне кружков (по каталогу) — закрывает.
5. Escape — закрывает.
6. Бургер поворачивается на 90° и меняется цвет при открытии.
7. Prefers-reduced-motion отключает анимацию.
8. На тёмной теме обводка и фон читаются.

## Out of scope

- Изменение нижней навигации (`BottomNavBar`) — она остаётся как есть.
- Изменение правых иконок в хедере (избранное, корзина) — не трогаем.
- Новая подсветка активного пункта меню в arc-виде — сейчас подсветки нет и в боковой панели, не добавляем.
